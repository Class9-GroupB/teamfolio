Project Learnings & Reflection

What I Built

As part of the Team4Tech Teamfolio project, I contributed to building a fully functional, production-ready web application on AWS. My primary focus was on infrastructure design, auto-scaling implementation, and ensuring high availability across multiple availability zones.

Testing Auto-Scaling

To validate our auto-scaling configuration, I conducted comprehensive load testing using k6. I simulated traffic patterns ramping from 50 to 200 virtual users over a 14-minute period. The key findings were:

  • Scale-Up Behavior: When CPU utilization exceeded 70%, new instances launched within 5 minutes and became healthy, demonstrating responsive horizontal scaling.
  • Performance Under Load: At 200 RPS, we maintained p95 latency under 800ms with less than 0.03% error rate.
  • Resilience Testing: Terminated one instance during peak load - the ALB immediately stopped routing traffic to it, and the remaining instances handled the load without user impact.
  • Scale-Down: When load decreased, the cooldown period prevented premature termination, ensuring stability.

Key Lesson Learned

The Power of Infrastructure as Code: Initially, I manually configured resources in the AWS Console, which was time-consuming and error-prone. After switching to Terraform, I could destroy and recreate the entire infrastructure in minutes with consistency and documentation built-in.

This taught me that while the learning curve for IaC tools is steeper upfront, the long-term benefits in repeatability, version control, and collaboration far outweigh the initial investment. Moving forward, I'll always start with IaC for any cloud project.

Technical Challenges Overcome

  • NAT Gateway Costs: Learned to balance high availability with budget constraints by implementing cost-saving strategies like scheduled scaling.
  • Health Check Tuning: Initially, health checks were too aggressive, causing unnecessary instance churn. Adjusted thresholds and grace periods for stability.
  • User Data Debugging: Troubleshooting failed instance launches taught me the importance of comprehensive logging and idempotent scripts.
  • Zero-Downtime Deployments: Implemented rolling updates with proper warmup periods and checkpoint delays to ensure seamless deployments.
